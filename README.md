# CoilLab

台股波段強勢股早期型態研究工具

這是一個台灣股市投資研究用的 MVP 工具，用來判斷單一股票是否具備「波段強勢股」的早期技術面條件。

目標不是直接產生買賣建議，而是把量價、均線、整理型態、相對強度與風險條件整理成可檢查的分數與報告，協助後續驗證規則是否有效。

## 目前功能

- 輸入上市股票代號，例如 `2330`，自動抓取個股區間日成交資料。
- 自動抓取 TAIEX 區間資料，用於計算相對強度。
- 籌碼資料不會在股票查詢時自動抓取，避免短時間大量 API 請求。
- 支援上傳 CSV，使用自備 OHLCV 資料分析單檔股票。
- 內建示範正樣本與負樣本，方便測試評分邏輯。
- 產生 `0-100` 技術分數與分類：
  - `80-100`：強勢候選
  - `60-79`：觀察中
  - `40-59`：條件不足
  - `<40`：不符合
- 顯示條件明細：
  - 趨勢結構
  - 整理型態
  - 量能狀態
  - 相對強度
  - 風險扣分
- 顯示交易決策輔助面板，區分可行動、觀察、偏延伸與避開。
- 顯示籌碼與基本面面板，先支援籌碼資料與基本面 unavailable 狀態。
- 籌碼面可檢查外資、投信、自營商、融資與融券狀態。
- 顯示技術失效位置、目標參考與估計報酬風險比。
- 顯示單檔股票訊號驗證回測，檢查高分訊號後 5/10/20/40/60 日表現。
- 顯示價格、20 日均線、60 日均線與成交量圖表。
- 顯示樣本驗證表，方便比較正負樣本分數。

## 資料來源

目前股票代號查詢使用 FinMind 區間資料：

- 上市個股日成交資訊（`TaiwanStockPrice`）
- TAIEX 總報酬指數歷史資料（`TaiwanStockTotalReturnIndex`）

每次股票代號查詢固定發出兩次 API 請求：一次個股日線與一次 TAIEX 日線。開發模式下透過 Vite dev server 的 `/finmind` proxy 轉發請求，避免瀏覽器 CORS 問題。

目前只支援 TWSE 上市股票。TPEx 上櫃股票尚未接入。

籌碼資料模組可保留給日後的手動載入功能；目前為了避免資料來源限流，股票查詢時不會自動取得籌碼資料。

## CSV 格式

CSV 至少需要以下欄位：

```csv
date,open,high,low,close,volume
```

範例：

```csv
date,open,high,low,close,volume
2026-01-02,100,103,99,102,1200000
2026-01-05,102,105,101,104,1350000
```

資料至少需要 120 筆交易日，否則無法完整計算中期技術條件。

## 使用方式

```bash

```

啟動開發伺服器：

```bash
npm run dev
```

打開瀏覽器：

```text
http://127.0.0.1:5173/
```

執行測試：

```bash
npm test
```

建立 production build：

```bash
npm run build
```

## 技術架構

- Vite
- React
- TypeScript
- Vitest
- Recharts
- Papa Parse

主要邏輯切分：

- `src/domain/validation.ts`：OHLCV 資料驗證
- `src/domain/indicators.ts`：均線、報酬、區間、量能指標
- `src/domain/scoring.ts`：強勢股規則評分
- `src/domain/report.ts`：分析報告組裝
- `src/domain/decision.ts`：交易決策輔助、技術失效位置與報酬風險估算
- `src/domain/backtest.ts`：單檔股票高分訊號驗證回測
- `src/domain/chipScoring.ts`：籌碼面規則評分與基本面 unavailable 狀態
- `src/domain/finMindData.ts`：FinMind 個股與 TAIEX 區間資料抓取/轉換
- `src/components/*`：畫面元件

## 限制

- 目前不是交易系統，也不提供買賣建議。
- 目前只評估技術面，尚未納入基本面與籌碼面。
- 目前只支援上市股票查詢，上櫃股票需後續接 TPEx。
- TWSE 資料未做除權息還原，長期比較可能受股本或價格調整影響。
- 本機開發使用 Vite proxy；若要正式部署，需要後端 API/proxy 或其他可跨域資料服務。
- 評分規則仍需用更多歷史強勢股與非強勢股樣本校準。
- 訊號驗證回測只驗證分數訊號，不模擬真實下單、滑價、手續費或部位控管。
- 基本面資料來源尚未正式接入，月營收與 EPS 目前先顯示為 unavailable。
- 籌碼資料目前不會自動載入，會顯示 unavailable，不中斷技術面分析。

## 後續方向

- 接入 TPEx 上櫃資料。
- 支援 Yahoo Finance CSV 欄位格式。
- 建立更多歷史正負樣本。
- 加入基本面條件，例如月營收、EPS、毛利率趨勢。
- 加入籌碼條件，例如外資、投信、融資券。
- 做全市場掃描與候選股排行。
- 增加訊號回測，檢查是否能在主升段前 2-6 週給出高分。
